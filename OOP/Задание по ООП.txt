Кратко описать, какими принципами ООП вы пользуетесь при разработке приложений (на каких этапах, обосновать)

1. Идея - выбор класса из которого при выполнении программы будет создан объект;
2. Проектирование - для выделение наиболее важных характеристик объекта применяеться метод абстракции. 
Обобщение информации о свойствах объектов, и отделение значимой информации от незначимой в контексте решаемой задачи. 
Метод абстракции позволяет выделить в иерархии объектов общую информацию: 
общий абстрактный тип объектов, общую характеристику, общий интерфейс и тд.
На основании этих абстрактных классов можно создавать в программе новые типы объектов.
3. Написание кода. Метод инкапсуляции позволяет решить несколько задачь, во-первых, разграничение однотипных объектов.
А, во-вторых, для исключения вмешательства в конструкцию и работу объекта, при этом атрибуты и поведение объекта объединяются в одном классе.
Для работы с объектом пользователям предоставляется открытый интерфейс.
При програмировании необходимо определить, какие атрибуты и методы будут доступны для открытого доступа, 
а какие будут являтся внутренней реализацией объекта и должны быть недоступны для изменений.
4. Написание кода. Метод наследования позволяет создавать новые классы с новыми функциональностями взяв за основу уже существующие классы, 
что позволяет существенно уменьшить объем написанного кода.
5. Проектирование/написание кода. Применяя метод полиморфизма можно из абстрактного класса (класса шаблона) можно создать производные классы, 
с дополнительной функцианальностью, но «ведущих себя» аналогично родственным, что, в свою очередь, позволяет достигнуть расширяемости и модифицируемости.
В дальнейшем это позволяет использовать один и тот же код и/или метод для обьектов разных классов и получать различные результаты.
